# 로한물류 웹사이트 제작 브리핑 (Claude Code용)

> 이 문서를 Claude Code에 프로젝트 사양으로 전달하세요. 처음엔 그대로 붙여넣고,
> 진행하면서 `CLAUDE.md` 로 옮겨두면 작업 내내 기준이 유지됩니다.

## 0. 작업 방식 (먼저 읽어주세요)

- **1인 개발자**가 혼자 만들고 혼자 유지보수합니다. 화려한 아키텍처보다 **단순하고 유지보수 쉬운 구조**를 우선하세요.
- 목표는 "동작하는 수준"이 아니라 **일반 상용 물류회사 웹사이트 수준의 완성도**입니다.
  반응형, 로딩/빈 상태/에러 처리, 통일된 디자인 시스템, 자연스러운 인터랙션을 기본으로 챙겨주세요.
- **진행 순서**: ① 프론트엔드(정적 사이트) 먼저 완성 → ② 배포는 그 다음에 별도로 논의합니다.
  지금 단계에서 배포/CI 설정은 만들지 마세요.
- **콘텐츠 채우는 방식**: 전체 틀과 컴포넌트를 **더미 데이터/플레이스홀더로 먼저** 완성하고,
  실제 텍스트·사진은 나중에 끼워 넣습니다. 그러니 각 데이터(차량/창고/거래처 등)는
  **데이터 객체로 분리**해서 나중에 값만 교체하기 쉽게 구성하세요.
- **큰 작업은 코드부터 짜지 말고 계획(폴더 구조·컴포넌트 목록·작업 순서)을 먼저 제시**하고,
  승인받은 뒤 한 단계씩 진행하세요.

## 1. 기술 스택

- **정적 사이트** (백엔드 없음). 서버에 데이터를 저장하거나 전송하지 않습니다.
- 개발자는 **Node.js · Docker · GitHub Actions · Render** 경험이 있고, 배포는 추후 **Render Static Site**로 할 예정입니다.
- 프레임워크는 1인 유지보수가 쉬운 선택지를 **근거와 함께 제안**해 주세요
  (예: 가벼운 빌드 셋업 또는 순수 HTML/CSS/JS). 결정은 승인 후 진행.
- 지도: **카카오맵 JS SDK(무료 키)** 또는 키가 필요 없는 **Leaflet** 중 제안. 정적 사이트에서 동작 가능해야 함.

## 2. 디자인 가이드

- 톤: **신뢰감·전문적·깔끔**, **블루 계열** 팔레트.
- 언어: **한국어 메인**, 세련된 느낌을 위한 **영문 디자인 요소** 병행 (예: 섹션 영문 라벨, 영문 슬로건).
- 참고 구조: 웅진북센(wjbooxenlogi.com) 스타일 — 히어로 배너 → 서비스 카드 → 회사소개 → 견적문의 CTA → 푸터.
  단, 내용은 본 문서 범위만큼만 가볍게. 색은 블루로 재구성.
- 로고: 개발 폴더에 **"로한물류 로고"** 파일명으로 저장되어 있음 → 헤더/푸터에 사용.

## 3. 사이트맵

1. **홈**
2. **회사소개** — 인사말 · 경영이념 · 연혁 · 기업개요 · 거래처(간단히)
3. **서비스** — 콜드체인 운송 · 3PL 물류대행 · 냉동/냉장 창고임대 + **전국 노선 지도**
4. **차량소개** — 사진 갤러리(카드형)
5. **창고소개** — 안성센터 · 진주센터 (※ 지도 넣지 않음)
6. **견적문의** — 폼 작성 후 요약 TXT 다운로드

## 4. 홈 화면 섹션 구성

1. **풀스크린 히어로** — 냉동탑차/창고 배경 + 콜드체인 강조 카피 + 영문 슬로건(예: COLD CHAIN LOGISTICS PARTNER) + "견적문의" CTA 버튼
2. **핵심 숫자 카운터** — 직영 차량 약 20대 · 물류센터 2개소(안성·진주) · 설립 2007 · 냉동/냉장/상온 겸용
3. **사업영역 카드 3개** — 콜드체인 운송 / 3PL 물류대행 / 냉동·냉장 창고임대
4. **차량 경쟁력 하이라이트** — 실내 스테인리스 마감 · 차량당 냉동기 2대 · 혹한기 히터 · 롤업도어·조르다
5. **전국 노선 지도 미리보기** → 서비스 페이지로 연결
6. **거래처 로고** (간단 배치, 로고 이미지는 추후 교체)
7. **견적문의 CTA 배너**
8. **푸터** — 본사 주소 · 노출 연락처 · 사업자 정보

## 9. 전국 노선 지도 (서비스 페이지)

위에서 아래로 내려오는 **문어발(허브-스포크) 구조**. 도시 중심 좌표로 마커와 선(polyline)을 그립니다.

### 마커
- **대구 본사**: 대구광역시 북구 한강로8길 13-13
- **칠곡물류IC**: 환적 거점(임대지, 회사 주소 아님) — 환적 허브로 표시
- **안성센터**: 경기 안성시 원곡면 원당로 233
- **진주센터**: 경남 진주시 진성면 동부로 1233
- (대구 본사와 칠곡물류IC는 지리적으로 가까움)

### 노선 노드(도시)
성남, 경기광주, 안성, 칠곡, 함안, 김해, 기장, 진주, 사천, 담양

### 노선 연결(선)
**영남 라인 — 경부선 + 중부내륙, 두 평행선:**
- 수도권 상단 집결: 성남–안성, 경기광주–안성 (안성이 수도권 거점)
- 안성 → 칠곡: **두 평행 노선(경부선 / 중부내륙)**
- 칠곡(환적 허브) → 함안 / 김해 / 기장 / 진주 (문어발 납품)

**전라 라인 — 서쪽으로 도는 별도 노선:**
- 사천·진주 → 담양 → 안성
- **진주는 칠곡(영남)·담양(전라) 양쪽에 모두 연결**
- ⚠️ **담양 ↔ 칠곡은 연결하지 말 것**

> 핵심: **안성 = 수도권/전라 거점**, **칠곡 = 영남 환적 허브**. 이 두 매듭에서 가지가 뻗는 형태.
> 정확 좌표는 도시 중심 + 4개 마커 주소를 지오코딩해서 잡습니다.

지도 위치: 홈에는 미리보기, 서비스 페이지에 노선 설명과 함께 본 지도. **창고소개 페이지엔 지도 없음.**

## 10. 견적문의 폼 + TXT 출력

**백엔드 없음.** 폼 제출 시 입력값을 요약해 **클라이언트에서 TXT 파일을 생성하여 다운로드**(Blob). 서버 전송/저장 없음.

### 필수 입력 항목
- 회사명
- 담당자명
- 연락처(휴대폰)
- 상차지(출발)
- 하차지(도착)
- 상차 희망시간
- 하차 희망시간
- 운송 시작 예정일
- 운송 주기 (1회성 / 정기)
- 화물 품목(종류)
- 온도대 (냉동 / 냉장 / 상온)
- 물량 (중량(톤) + 파렛트 수)
- 상차방법 (지게차 상차 여부)

### 선택 입력 항목
- 이메일
- 포장형태
- 희망 차량 톤수 (5~11톤)
- 기타 요청사항

### 기타
- **개인정보 수집·이용 동의 체크박스** (필수 — 체크해야 제출 가능)
- 필수 항목 미입력 시 사용자 친화적 검증 메시지 표시
- TXT 출력: 위 항목을 섹션별로 정리 + 맨 아래 **작성 일시**
- 파일명 예시: `로한물류_견적문의_YYYYMMDD.txt`
